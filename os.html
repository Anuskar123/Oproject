📐 Program 1: Rectangle Area Calculator

Requirements
20% of total grade
  • Prompt user for width and height in centimeters
  • Calculate area in square centimeters and square inches (1 inch = 2.54 cm)
  • Use regular expressions for input validation
  • Accept both floating point (3.142) and integer (12) input formats
  • Option to quit the program or enter new data
  • Use bc for floating-point calculations
  • Comprehensive error checking for all inputs
  • Must handle edge cases and invalid inputs gracefully
Implementation Structure
#!/bin/bash # area.sh - Rectangle Area Calculator # Author: [Your Name] # Student ID: [Your ID] # Date: [Current Date] # Description: Calculates rectangle area in cm² and square inches # Constants INCH_TO_CM=2.54 # Function to validate numeric input validate_number() { local input="$1" if [[ $input =~ ^[0-9]+\.?[0-9]*$ ]] && [[ $input != "0" ]]; then return 0 else return 1 fi } # Function to calculate area calculate_area() { local width="$1" local height="$2" # Calculate area in cm² local area_cm=$(echo "scale=2; $width * $height" | bc) # Convert to square inches local area_inches=$(echo "scale=2; $area_cm / ($INCH_TO_CM * $INCH_TO_CM)" | bc) echo "Area in square centimeters: $area_cm cm²" echo "Area in square inches: $area_inches in²" } # Main program loop main() { while true; do echo "=== Rectangle Area Calculator ===" # Get width while true; do read -p "Enter width in cm: " width if validate_number "$width"; then break else echo "Error: Please enter a valid positive number" fi done # Get height while true; do read -p "Enter height in cm: " height if validate_number "$height"; then break else echo "Error: Please enter a valid positive number" fi done # Calculate and display results calculate_area "$width" "$height" # Ask if user wants to continue read -p "Continue? (y/n): " continue if [[ $continue != "y" && $continue != "Y" ]]; then break fi done echo "Thank you for using the Rectangle Area Calculator!" } main "$@"
Key Features
Input Validation +

Use regular expressions to validate numeric input:

if [[ $input =~ ^[0-9]+\.?[0-9]*$ ]]; then # Valid number else # Invalid input fi
Floating Point Calculations +

Use bc for precise floating-point arithmetic:

area_cm=$(echo "scale=2; $width * $height" | bc) area_inches=$(echo "scale=2; $area_cm / ($INCH_TO_CM * $INCH_TO_CM)" | bc)

Tax Calculation Logic

  • Basic rate: £12,501-£50,000 (20% tax)
  • Higher rate: £50,001-£150,000 (40% tax)
# Example tax calculation function calculate_tax() { local gross_salary=$1 local tax=0 if (( $(echo "$gross_salary > 50000" | bc -l) )); then # Higher rate tax on amount over £50,000 local higher_rate_amount=$(echo "$gross_salary - 50000" | bc -l) local higher_tax=$(echo "$higher_rate_amount * 0.40" | bc -l) # Basic rate tax on £37,500 (£50,000 - £12,500) local basic_tax=$(echo "37500 * 0.20" | bc -l) tax=$(echo "$higher_tax + $basic_tax" | bc -l) elif (( $(echo "$gross_salary > 12500" | bc -l) )); then # Basic rate tax only local taxable_amount=$(echo "$gross_salary - 12500" | bc -l) tax=$(echo "$taxable_amount * 0.20" | bc -l) fi echo $tax }

🃏 Program 4: Blackjack Game (blackjack.sh)

Requirements Analysis
  • Core Rules: Implement full blackjack rules as described in assignment
  • Players: Support one dealer and at least one player
  • Data Structures: Use arrays for cards, hands, and deck management
  • Card Management: Remove dealt cards from deck to prevent duplicates
  • Functions: Use functions for game logic and modularity
  • User Interface: Make the program user-friendly with clear prompts
  • Game Flow: Implement proper turn-based gameplay
  • Scoring: Handle Ace values (1 or 11) correctly
  • Special Cases: Handle blackjack, bust, push scenarios
Game Structure Planning
# Card deck representation declare -a DECK declare -a PLAYER_HAND declare -a DEALER_HAND # Game functions needed: # - initialize_deck() # - shuffle_deck() # - deal_card() # - calculate_hand_value() # - player_turn() # - dealer_turn() # - determine_winner()
Key Game Logic
  1. Card Values: 2-10 = face value, J/Q/K = 10, A = 1 or 11 (player's choice)
  2. Blackjack: Ace + 10-value card = 21 (automatic win unless dealer also has blackjack)
  3. Dealer Rules: Must hit on 16 or less, must stand on 17 or more
  4. Win Conditions: Player closer to 21 without busting wins
  5. Bust: Total over 21 results in automatic loss
  6. Push: Same hand value as dealer results in tie
  7. Five Card Charlie: Optional bonus for 5 cards without busting
Game Flow Implementation
  1. Setup: Initialize deck, shuffle cards, get number of players
  2. Initial Deal: Deal 2 cards to each player and dealer (dealer has 1 face down)
  3. Check for Blackjack: Check if any player or dealer has natural 21
  4. Player Turns: Each player decides to hit or stand
  5. Dealer Turn: Dealer plays according to rules (hit on 16, stand on 17)
  6. Determine Winners: Compare hands and determine outcomes
  7. Score Tracking: Optional points system for multiple rounds
Key Terms & Definitions
  • Hit: Request another card
  • Stand/Stick: Keep current hand, no more cards
  • Bust: Hand total exceeds 21
  • Push: Tie between player and dealer
  • Blackjack: Natural 21 with first two cards
  • Five Card Charlie: Five cards without busting (optional bonus)
  • Hole Card: Dealer's face-down card
